# OpenSpider GitHub部署指南

## 快速部署

### 1. 创建GitHub Personal Access Token

1. 访问: https://github.com/settings/tokens
2. 点击 "Generate new token (classic)"
3. 设置名称: "OpenSpider Deploy"
4. 勾选权限:
   - [x] repo - Full control of private repositories
   - [x] admin:org - Full control of orgs and teams, read org assets
5. 点击 "Generate token"
6. **保存好token**，只显示一次

### 2. 设置环境变量并部署

```bash
export GITHUB_USER="你的GitHub用户名"
export GITHUB_TOKEN="你的token"

# 运行部署脚本
cd /root/.openclaw/workspace
node deploy.js
```

### 3. 查看结果

部署成功后，访问:
```
https://github.com/你的用户名/openspider
```

## 手动部署

如果脚本有问题，可以手动部署:

```bash
cd /root/.openclaw/workspace

# 1. 创建仓库
curl -X POST \
  -H "Authorization: token YOUR_TOKEN" \
  -H "Accept: application/vnd.github.v3+json" \
  https://api.github.com/user/repos \
  -d '{"name":"openspider","description":"OpenSpider Plan - NRT组织建设方案","private":false}'

# 2. 添加远程并推送
git remote add origin "https://YOUR_TOKEN@github.com/YOUR_USER/openspider.git"
git branch -M main
git push -u origin main
```

## 问题排查

### Q: token权限不足
A: 确保勾选了 `repo` 权限

### Q: 仓库已存在
A: 可以删除已存在的仓库，或者使用不同的仓库名（需要修改脚本中的REPO_NAME）

### Q: 推送失败
A: 检查网络连接，确保可以访问 github.com
