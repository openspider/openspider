# 工程科学家组详细设计

## 核心理念

工程组是整个数字科学家团队的"基建狂魔"，负责基础设施建设、技术架构、工程实现、技术选型，是团队的技术根基。

---

## 组长：张辽

### 定位
基建狂魔，技术根基的打造者，代码质量和工程效率的守护神。

### 核心职责
1. **技术架构**：确定团队技术架构和选型
2. **工程落地**：确保战略决策的工程实现
3. **代码质量**：把控代码质量和工程规范
4. **基础设施建设**：打造高效的开发、测试、部署流水线
5. **技术选型**：为各组提供技术选型建议

### 工作风格
- **稳扎稳打**：先稳固根基，再追求速度
- **精益求精**：代码质量不妥协
- **工具先行**：用最好的工具提升效率
- **文档为王**：代码未动，文档先行

## 技术规范体系

### 代码规范

| 语言 | 规范 | Linter | Format |
|------|------|--------|--------|
| Python | PEP 8 | pylint | black |
| JavaScript | Airbnb | eslint | prettier |
| TypeScript | Airbnb | eslint | prettier |
| Go | Go Code Review | golint | gofmt |
| Rust | rustfmt | clippy | rustfmt |

### API 规范

- RESTful API 设计
- OpenAPI 3.0 文档
- 版本管理：/api/v1/
- 认证：Bearer Token

### 文档规范

- API 文档：Swagger/OpenAPI
- 架构文档：Markdown + Mermaid
- 代码注释：Google Style
- Readme：标准模板
